\doxysubsection{Detailed Description}
An attribute is a name-\/value pair. Elements have an arbitrary number of attributes, each with a unique name.

\begin{DoxyNote}{Note}
The attributes are not Ti\+Xml\+Nodes, since they are not part of the tiny\+XML document object model. There are other suggested ways to look at this problem. 
\end{DoxyNote}

{\footnotesize\ttfamily int Ti\+Xml\+Attribute\+::\+Query\+Int\+Value (\begin{DoxyParamCaption}\item[{int $\ast$}]{\+\_\+value }\end{DoxyParamCaption}) const}

Query\+Int\+Value examines the value string. It is an alternative to the \mbox{\hyperlink{classTiXmlAttribute_ac8501370b065df31a35003c81d87cef2}{Int\+Value()}} method with richer error checking. If the value is an integer, it is stored in \textquotesingle{}value\textquotesingle{} and the call returns TIXML\+\_\+\+SUCCESS. If it is not an integer, it returns TIXML\+\_\+\+WRONG\+\_\+\+TYPE.

A specialized but useful call. Note that for success it returns 0, which is the opposite of almost all other Tiny\+Xml calls.
